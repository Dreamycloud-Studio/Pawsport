Pawsport — React Native Monorepo Setup
Implementation instructions for coding AI — March 22, 2026

OVERVIEW AND GOAL

Set up a pnpm monorepo containing the existing Pawsport React web app and a new React Native (Expo) mobile app for iOS and Android. Both apps share business logic, API calls, TypeScript types, and utility functions via a shared core package. Do not rewrite or move the existing web app — integrate it into the monorepo structure around it.

Target structure:
pawsport/
├── apps/
│   ├── web/              (existing React app, moved here)
│   └── mobile/           (new React Native + Expo app)
├── packages/
│   └── core/             (new shared TypeScript library)
├── package.json          (workspace root)
├── pnpm-workspace.yaml
└── turbo.json

packages/core/src/api/rag.ts — RAG query function used by both apps:

import { RagResponse } from "../types";

export async function queryRegulations(
  question: string,
  destinationCountry: string,
  petType: string,
  apiBaseUrl: string
): Promise<RagResponse> {
  const response = await fetch(`${apiBaseUrl}/api/rag/query`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ question, destinationCountry, petType }),
  });
  if (!response.ok) {
    throw new Error(`RAG query failed: ${response.statusText}`);
  }
  return response.json();
}

packages/core/src/utils/timeline.ts — shared timeline helpers:

export interface TimelineEvent {
  id: string;
  title: string;
  description: string;
  dueDate: string;
  topic: string;
  urgent: boolean;
}

export function calculateTimeline(
  travelDate: string,
  requirements: string[]
): TimelineEvent[] {
  const travel = new Date(travelDate);
  const events: TimelineEvent[] = [];

  if (requirements.includes("rabies_vaccine")) {
    const vaccineDeadline = new Date(travel);
    vaccineDeadline.setDate(vaccineDeadline.getDate() - 180);
    events.push({
      id: "rabies-vaccine",
      title: "Rabies vaccination",
      description: "Must be administered at least 180 days before travel for titer test to be valid.",
      dueDate: vaccineDeadline.toISOString().split("T")[0],
      topic: "vaccine",
      urgent: daysUntil(vaccineDeadline) < 14,
    });
  }

  if (requirements.includes("health_certificate")) {
    const certDeadline = new Date(travel);
    certDeadline.setDate(certDeadline.getDate() - 10);
    events.push({
      id: "health-cert",
      title: "Health certificate",
      description: "Issued by accredited vet, valid within 10 days of travel.",
      dueDate: certDeadline.toISOString().split("T")[0],
      topic: "health_certificate",
      urgent: daysUntil(certDeadline) < 7,
    });
  }

  return events.sort((a, b) => a.dueDate.localeCompare(b.dueDate));
}

function daysUntil(date: Date): number {
  return Math.ceil((date.getTime() - Date.now()) / (1000 * 60 * 60 * 24));
}

import { useState } from "react";
import { View, Text, TextInput, Button, ScrollView, StyleSheet } from "react-native";
import { queryRegulations, RagResponse } from "@pawsport/core";

export default function AssistantScreen() {
  const [question, setQuestion] = useState("");
  const [country, setCountry] = useState("JP");
  const [petType, setPetType] = useState("cat");
  const [response, setResponse] = useState<RagResponse | null>(null);
  const [loading, setLoading] = useState(false);

  async function handleAsk() {
    setLoading(true);
    try {
      const result = await queryRegulations(
        question,
        country,
        petType,
        process.env.EXPO_PUBLIC_API_BASE_URL!
      );
      setResponse(result);
    } finally {
      setLoading(false);
    }
  }

  return (
    <ScrollView style={styles.container}>
      <TextInput
        style={styles.input}
        placeholder="Ask about pet travel regulations..."
        value={question}
        onChangeText={setQuestion}
        multiline
      />
      <Button title={loading ? "Asking..." : "Ask"} onPress={handleAsk} disabled={loading} />
      {response && (
        <View style={styles.answer}>
          <Text style={styles.answerText}>{response.answer}</Text>
          {response.sources.map((s) => (
            <Text key={s.url} style={styles.source}>Source: {s.name}</Text>
          ))}
        </View>
      )}
    </ScrollView>
  );
}

const styles = StyleSheet.create({

Step 8: Push notification setup for deadline reminders.

apps/mobile/lib/notifications.ts:

import * as Notifications from "expo-notifications";
import { TimelineEvent } from "@pawsport/core";

export async function requestNotificationPermissions(): Promise<boolean> {
  const { status } = await Notifications.requestPermissionsAsync();
  return status === "granted";
}

export async function scheduleDeadlineNotifications(events: TimelineEvent[]): Promise<void> {
  await Notifications.cancelAllScheduledNotificationsAsync();
  for (const event of events) {
    const triggerDate = new Date(event.dueDate);
    triggerDate.setDate(triggerDate.getDate() - 7);
    if (triggerDate > new Date()) {
      await Notifications.scheduleNotificationAsync({
        content: {
          title: event.urgent ? "Urgent: " + event.title : event.title,
          body: event.description,
          data: { eventId: event.id },
        },
        trigger: { date: triggerDate },
      });
    }
  }
}

PART 6 — KEY IMPLEMENTATION NOTES FOR AI

1. The @pawsport/core package uses workspace:* protocol — pnpm resolves this automatically. Do not publish it to npm.

2. Metro bundler (React Native) does not support TypeScript path aliases out of the box. The tsconfig paths above work because Expo handles this via babel-plugin-module-resolver. Do not remove the paths config.

3. All environment variables in the mobile app must be prefixed EXPO_PUBLIC_ to be accessible at runtime. Server-side secrets (ANTHROPIC_API_KEY, DATABASE_URL) never go in the mobile app — they stay in Vercel serverless functions.

4. Supabase auth tokens on mobile should be stored in expo-secure-store (hardware-backed keychain/keystore), not AsyncStorage (unencrypted).

5. The queryRegulations function in core hits the Vercel API, not Supabase or the Anthropic API directly. Mobile apps never call LLM APIs directly — always proxy through the Vercel backend.

6. For offline checklist support, use expo-sqlite to mirror checklist items locally and sync on reconnect. The schema mirrors the trips and checklist_items tables in Supabase.

7. When adding new shared logic, always put it in packages/core/src/ and export from index.ts. Never copy-paste the same function into both apps.
